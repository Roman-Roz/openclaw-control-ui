# 🦞 OpenClaw Cyberpunk Control Panel

**Неоновая панель управления OpenClaw в стиле киберпанк**

![Cyberpunk Interface](assets/preview.png)

## 🚀 Быстрый старт

### Требования
- **Node.js 16+** (рекомендуется 18+)
- **OpenClaw Gateway** (запущен на `http://127.0.0.1:18789`)
- Современный браузер (Chrome 90+, Firefox 88+, Edge 90+)

### Установка

1. **Клонируйте/скачайте проект**
   ```bash
   # Если у вас есть git
   git clone <репозиторий>
   cd openclaw-cyberpunk-panel
   
   # Или просто распакуйте архив
   unzip openclaw-cyberpunk-panel.zip
   cd openclaw-cyberpunk-panel
   ```

2. **Установите зависимости**
   ```bash
   npm install
   ```

3. **Запустите панель**
   ```bash
   npm start
   ```

4. **Откройте в браузере**
   ```
   http://localhost:3000
   ```

## 🔧 Конфигурация

### Изменение порта
По умолчанию панель работает на порту 3000. Чтобы изменить порт:

1. Отредактируйте `server.js`:
   ```javascript
   const PORT = 8080; // ваш порт
   ```

2. Или используйте переменную окружения:
   ```bash
   PORT=8080 npm start
   ```

### Подключение к другому Gateway
Если ваш OpenClaw Gateway работает на другом адресе:

1. Отредактируйте `server.js`:
   ```javascript
   target: 'http://ваш-адрес:ваш-порт',
   ```

2. Или используйте переменную окружения:
   ```bash
   GATEWAY_URL=http://192.168.1.100:18789 npm start
   ```

1. **Дашборд (Dashboard)**
   - Общий статус системы
   - Активные агенты
   - Использование моделей
   - Быстрый терминал

2. **Агенты (Agents)**
   - Просмотр всех агентов
   - Запуск/остановка агентов
   - Настройка параметров

3. **Модели (Models)**
   - Переключение между моделями
   - Информация о стоимости
   - Настройка fallback-цепочки

4. **Терминал (Terminal)**
   - Полноценный веб-терминал
   - История команд
   - Автодополнение

5. **Логи (Logs)**
   - Реальное обновление логов Gateway
   - Фильтрация по уровню (info, warn, error)
   - Поиск по тексту

6. **Конфигурация (Config)**
   - Безопасное редактирование настроек
   - Валидация JSON
   - Резервное копирование

## 🔍 Устранение неполадок

### Панель не подключается к Gateway

1. **Проверьте статус Gateway:**
   ```bash
   openclaw gateway status
   ```

2. **Убедитесь, что Gateway слушает на правильном порту:**
   ```bash
   netstat -tlnp | grep 18789
   ```

3. **Проверьте настройки прокси в `server.js`**

### Ошибка "Connection refused"

1. **Gateway может быть недоступен из-за firewall:**
   ```bash
   # Проверьте доступность
   curl http://127.0.0.1:18789/status
   ```

2. **Измените настройки bind в OpenClaw:**
   ```json
   "gateway": {
     "bind": "loopback"  // или "lan"
   }
   ```

### Медленная работа интерфейса

1. **Используйте hardware acceleration в браузере**
2. **Отключите тяжёлые анимации в настройках панели**
3. **Увеличьте таймауты в `script.js`**

## 🔒 Безопасность

### Рекомендации

1. **Не разворачивайте панель в публичной сети**
2. **Используйте HTTPS в production**
3. **Настройте аутентификацию через Gateway token**
4. **Регулярно обновляйте зависимости**

### Настройка аутентификации

Добавьте в `server.js`:

```javascript
// Базовая HTTP аутентификация
app.use('/admin', (req, res, next) => {
  const auth = { login: 'admin', password: 'ваш-пароль' };
  const b64auth = (req.headers.authorization || '').split(' ')[1] || '';
  const [login, password] = Buffer.from(b64auth, 'base64').toString().split(':');
  
  if (login === auth.login && password === auth.password) {
    return next();
  }
  res.set('WWW-Authenticate', 'Basic realm="OpenClaw Panel"');
  res.status(401).send('Требуется аутентификация');
});
```
